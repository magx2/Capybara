= Module Duration

== Functions

=== fun from_iso_8601(iso: xref:../../capy/lang/String.adoc#type-String[String]): xref:../../capy/lang/Result.adoc#type-Result[Result][xref:#type-Duration[Duration]]

Parses an ISO 8601 duration string.

Accepted inputs:
- symbolic durations like `P3Y6M4DT12H30M5S`
- week-only durations like `P2W`
- combined basic form like `P00030604T123005`
- combined extended form like `P0003-06-04T12:30:05`

Parsing rules:
- at least one component must be present
- `W` cannot be mixed with other components
- components must appear in ISO 8601 order
- only integer values are supported

Returns `Error` with a readable message when the input is invalid.

Example:
```cfun
from_iso_8601("PT90S")
from_iso_8601("P2W")
from_iso_8601("P0003-06-04T12:30:05")
```
These return a 90-second date duration, a two-week duration, and a combined
extended duration respectively.

== Types

[[type-DateDuration]]
=== data DateDuration

Duration expressed with calendar and clock components.

Supported ISO 8601 units:
- `Y` years
- `M` months
- `D` days
- `H` hours
- `M` minutes in the time part
- `S` seconds

Example:
```cfun
DateDuration { years: 3, months: 6, days: 4, hours: 12, minutes: 30, seconds: 5 }
```

* `hours`: int
* `minutes`: int

[[type-Duration]]
=== union Duration

[ISO 8601 - Durations](https://en.wikipedia.org/wiki/ISO_860101)

Represents a calendar/time duration using integer components.

Supported fields:
- `years`, `months`, `weeks`, `days`
- `hours`, `minutes`, `seconds`

Notes:
- `weeks` is treated as a standalone ISO 8601 duration unit.
- The parser accepts integer-only values.
- Fractional values like `PT0.5S` are not supported.

`Duration` is a sum type:
- `DateDuration` for year/month/day/time durations
- `WeekDuration` for ISO week-only durations like `P2W`

Example:
```cfun
from_iso_8601("P3Y6M4DT12H30M5S")
```
Returns a `DateDuration` with date and time components populated.

* xref:#type-DateDuration[DateDuration]
* xref:#type-WeekDuration[WeekDuration]

==== Functions

===== fun Duration.days(): int

Returns day component (`weeks * 7` for week durations).


===== fun Duration.hours(): int

Returns hour component (`0` for week durations).


===== fun Duration.minutes(): int

Returns minute component (`0` for week durations).


===== fun Duration.months(): int

Returns month component (`0` for week durations).


===== fun Duration.negate(): xref:#type-Duration[Duration]

Returns this duration with all components negated.
Returns the same duration with every component sign inverted.

Example: `DateDuration { years: 0, months: 0, days: 1, hours: 0, minutes: 0, seconds: 0 }.negate()`
returns a negative one-day duration.


===== fun Duration.seconds(): int

Returns second component (`0` for week durations).


===== fun Duration.to_iso_8601(): xref:../../capy/lang/String.adoc#type-String[String]

Formats this duration as a canonical ISO 8601 string.

Examples:
```cfun
ZERO.to_iso_8601() == "PT0S"
DateDuration { years: 3, months: 6, days: 4, hours: 12, minutes: 30, seconds: 5 }.to_iso_8601() == "P3Y6M4DT12H30M5S"
WeekDuration { weeks: 2 }.to_iso_8601() == "P2W"
```

The output uses the symbolic ISO 8601 form and omits zero-valued components.


===== fun Duration.years(): int

Returns year component (`0` for week durations).


[[type-WeekDuration]]
=== data WeekDuration

ISO 8601 week-only duration.

This variant is used for strings like `P2W`.
It is intentionally separate from `DateDuration`, because ISO week durations
are not mixed with year/month/day/time components in this API.

Example:
```cfun
WeekDuration { weeks: 2 }
```

* `weeks`: int
